# 变更提案：recipe-create-familiarity-v1

## 为什么

当前新建菜谱页仅暴露“默认加入想做清单（want_to_make）”，没有显式暴露“没做过/做过/常做（familiarity）”选择，导致用户在创建时无法直接设置熟悉度，并与“全部菜谱”筛选语义不够对齐。

## 变更内容

- 在新建菜谱页新增 `familiarity` 三态选择：
  - 没做过（`new`）
  - 做过（`done`）
  - 常做（`frequent`）
- 保留并继续使用“默认加入想做清单（want_to_make）”开关，保持与熟悉度独立。
- 同步补充单测，覆盖默认值与显式选择场景。

## 影响范围

- 前端表单与测试变更。
- 不涉及 Supabase schema 变更（字段与约束已存在）。
