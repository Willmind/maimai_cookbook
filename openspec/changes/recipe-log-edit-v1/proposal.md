# 变更提案：recipe-log-edit-v1

## 为什么

当前 MVP 支持新建菜谱与新建做饭记录，但用户在创建后无法修改，导致录入错误或后续复盘补充只能新建，实际使用摩擦较高。

## 变更内容

- 新增菜谱编辑能力：
  - 增加菜谱编辑入口与路由
  - 允许修改菜谱主要字段（名称、来源、简介、封面、食材、做法、下次注意、想做状态）
- 新增做饭记录编辑能力：
  - 做饭记录列表项可点击进入编辑
  - 允许修改记录字段（日期、结果、照片、记录、变化、下次注意）
- 数据层补充 `update` 能力，保持“组件通过 repository 访问数据”的约束。

## 影响范围

- 前端路由、页面、repository 接口与实现、单测。
- Supabase schema 本次不变更；现有字段与 `update` 策略可满足需求。
