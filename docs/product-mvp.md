# 麦麦手记 MVP 产品文档

## 1. 产品定位

麦麦手记是一个个人厨房经验记录应用。它不是公开菜谱平台，也不是复杂的菜单计划工具，而是帮助用户把“想做的菜、已经做过的菜、每次做饭后的经验”沉淀下来。

第一版重点解决三件事：

- 想做什么菜，可以先记下来。
- 做过一次后，可以留下这次的结果、照片和下次注意。
- 以后找菜时，可以快速看到菜谱本身和相关做饭记录。

## 2. MVP 范围

### 包含

- 首页：搜索、最近做过、想做清单。
- 全部菜谱页：查看所有菜谱，并用轻量筛选定位菜谱。
- 新建菜谱：只要求菜名必填，其他信息可选。
- 菜谱详情：展示菜谱档案、想做状态、熟悉度、做饭记录列表。
- 记录一次：从菜谱详情进入，为指定菜谱新增一次做饭记录。
- 图片上传：菜谱封面和做饭记录照片第一版都只支持 1 张。
- 空状态和无搜索结果状态。

### 暂不包含

- 随机一道。
- 全部做饭记录独立列表。
- 全局新增做饭记录入口。
- 动态新增标签。
- 多图上传。
- 菜单计划、购物清单、公开分享、多人协作。
- 详情页的“沉淀经验”独立动作。

## 3. 核心对象

### 菜谱 `recipes`

菜谱是独立档案，用来描述一道菜“下次可以怎么做”。一个菜谱可以没有做饭记录，也可以有多条做饭记录。

建议字段：

| 字段 | 类型 | 规则 |
| --- | --- | --- |
| `id` | uuid | 主键 |
| `name` | text | 必填，唯一性第一版不强制 |
| `source` | text | 可选，来源 / 灵感 |
| `description` | text | 可选，简介 |
| `cover_image_path` | text | 可选，最多 1 张封面 |
| `ingredients` | text | 可选，第一版可按多行文本存 |
| `method` | text | 可选，做法 |
| `next_note` | text | 可选，小贴士 / 下次注意 |
| `familiarity` | text | 可选，默认 `new` |
| `want_to_make` | boolean | 默认 `true` |
| `tags` | text[] | 可选，来自前端固定标签 |
| `created_at` | timestamptz | 创建时间 |
| `updated_at` | timestamptz | 更新时间 |

`familiarity` 建议枚举：

- `new`：没做过。
- `done`：做过。
- `frequent`：常做。

`want_to_make` 和 `familiarity` 互相独立。做过或常做的菜也可以继续保留在想做清单里，因为“想做”表达的是接下来还想再做，而不是是否做过。

### 做饭记录 `cooking_logs`

做饭记录是一次真实下厨后的记录，必须关联到一个菜谱。第一版不支持没有菜谱时直接记录一次。

建议字段：

| 字段 | 类型 | 规则 |
| --- | --- | --- |
| `id` | uuid | 主键 |
| `recipe_id` | uuid | 必填，关联 `recipes.id` |
| `cooked_at` | date | 可选，默认今天 |
| `result` | text | 可选 |
| `note` | text | 可选，这次记录 |
| `changes` | text | 可选，和上次比改了什么 |
| `next_note` | text | 可选，下次要注意什么 |
| `photo_path` | text | 可选，最多 1 张照片 |
| `created_at` | timestamptz | 创建时间 |

`result` 建议枚举：

- `good`：好吃。
- `ok`：一般。
- `failed`：翻车。

## 4. 页面与入口

### 首页

首页是高频入口，不承担完整管理职责。

首页包含：

- 顶部操作：`全部菜谱`、`记一道菜`。
- 搜索框：搜索菜名、食材、标签、做法、做饭记录内容。
- 最近做过：展示最近的做饭记录，建议 3 条。
- 想做：展示 `want_to_make = true` 的菜谱，建议 3 条。

首页不包含：

- 随机一道。
- 全部菜谱列表。
- 全部做饭记录列表。
- 排序链接。

### 全部菜谱页

全部菜谱是独立页面，后续路由可设为 `/recipes`。

页面包含：

- 搜索框。
- 轻量筛选：`全部`、`想做`、`没做过`、`做过`、`常做`。
- 菜谱列表。

筛选保持轻量，不做标签筛选面板。标签只参与搜索和列表展示。

### 新建菜谱页

后续路由可设为 `/recipes/new`。

字段规则：

- `菜名` 必填。
- `来源 / 灵感` 可选。
- `简介` 可选。
- `封面图` 可选，最多 1 张。
- `食材` 可选。
- `做法` 可选。
- `小贴士 / 下次注意` 可选。
- `熟悉度` 可选，默认 `没做过`。
- `想做清单` 默认加入，可取消。
- `标签` 可选，来自前端固定标签。

保存后进入菜谱详情页。

### 菜谱详情页

后续路由可设为 `/recipes/:id`。

页面包含：

- 菜名、简介、来源、封面图。
- 主要动作：`记一次`。
- 想做状态按钮，例如 `已想做` / `加入想做`。
- 熟悉度与标签展示。
- 下次注意。
- 食材、做法。
- 做饭记录列表。

做饭记录列表展示：

- 日期。
- 结果。
- 这次记录。
- 下次注意。
- 如果有照片，展示缩略图；如果没有照片，不展示空占位。

详情页不保留单独的 `新增记录` 链接，避免和顶部 `记一次` 重复。

### 记录一次页

后续路由可设为 `/recipes/:id/logs/new`。

进入前提：

- 必须从菜谱详情进入。
- 必须已经存在菜谱。

字段规则：

- `这次做的是` 只读，从菜谱详情带入，不可修改。
- `日期` 可选，默认今天。
- `成品照片` 可选，最多 1 张。
- `这次做得怎么样` 可选。
- `这次记录` 可选。
- `和上次比改了什么` 可选。
- `下次要注意什么` 可选。

保存后回到菜谱详情，并在做饭记录列表中看到新记录。

## 5. 搜索规则

搜索以“先帮用户找到可以做的菜”为目标，不做复杂排序配置。

建议搜索结果分两组：

- 菜谱。
- 做饭记录。

菜谱排序建议：

1. 菜名命中。
2. 标签、食材、做法命中。
3. 有做饭记录的菜谱优先。
4. 最近更新的菜谱优先。

做饭记录排序建议：

1. 记录内容命中。
2. 最近做过优先。

无搜索结果时：

- 提示用户换关键词。
- 提供 `记一道新菜` 入口。

## 6. 图片上传规则

第一版只支持单图，降低实现复杂度。

菜谱封面：

- 最多 1 张。
- 可上传、预览、更换、删除。

做饭记录照片：

- 最多 1 张。
- 可上传、预览、更换、删除。

上传状态需要覆盖：

- 上传中：显示压缩 / 上传进度。
- 上传成功：显示缩略图、文件名、`更换`、`删除`。
- 上传失败：显示失败原因、`重试`、`移除`。

Supabase Storage 建议分两个目录：

- `recipe-covers/{user_id}/{recipe_id}.webp`
- `cooking-log-photos/{user_id}/{log_id}.webp`

## 7. 固定标签

第一版不做动态标签管理。标签由前端常量维护，用户只能从已有标签中选择。

初始标签可以包括：

- 家常。
- 快手。
- 下饭。
- 早餐。
- 招待。
- 清淡。
- 汤。
- 晚饭。

后续如果标签数量明显变多，再考虑独立标签表。

## 8. Supabase 实现建议

### 表关系

```text
recipes 1 ── n cooking_logs
```

`cooking_logs.recipe_id` 必须引用 `recipes.id`。

### 权限

如果第一版只有个人使用，可以先按单用户思路开发。若后续要登录，建议所有表增加 `user_id` 并开启 Row Level Security。

### 更新规则

- 新建菜谱时写入 `recipes`。
- 新建做饭记录时写入 `cooking_logs`。
- 新建做饭记录后，可以选择同步更新菜谱的 `updated_at`。
- 是否自动把 `familiarity` 从 `new` 改成 `done`，第一版建议先不自动改，避免隐藏规则让用户困惑；后续可在详情页提供手动调整。

## 9. 验收标准

- 用户只填写菜名，也可以成功创建菜谱。
- 新建菜谱默认加入想做清单，用户可以取消。
- 没有菜谱时，不能记录一次做饭。
- 用户只能从菜谱详情进入记录一次。
- 记录一次页面中的菜名是只读的。
- 首页最近做过展示的是做饭记录，不是菜谱。
- 首页想做展示的是菜谱，不是做饭记录。
- 全部菜谱页可以通过搜索和 5 个轻量筛选找到菜谱。
- 搜索结果先展示菜谱，再展示做饭记录。
- 菜谱详情可以看到来源、简介、想做状态、熟悉度、食材、做法、下次注意和做饭记录。
- 菜谱详情的做饭记录有照片时显示缩略图，没有照片时不显示空图框。
- 菜谱封面和做饭记录照片都只允许 1 张，并支持更换和删除。

## 10. 后续迭代

可以放到第二版之后再考虑：

- 详情页“沉淀经验”动作：把多次做饭记录总结回菜谱。
- 多图上传。
- 全部做饭记录独立页面。
- 更细的筛选和排序。
- 动态标签管理。
- 菜单计划和购物清单。
- 数据导出。
- 登录、多设备同步和 RLS 权限完善。
